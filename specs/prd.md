# AIICC — AI 商科辅导平台 PRD

> 让每个学员都有一位 AI 创新导师 | 心光教育出品

---

## 1. 产品定位

### 使命
用 AI 放大教练能力，让每个参加 ICC 大赛的学员都能获得高质量、个性化的创新项目辅导。

### 一句话描述
AIICC 是一个 AI 驱动的创新项目辅导平台——学员通过结构化工作流 + 多角色 AI 对话，在引导中完成从创意到交付的完整 ICC 参赛项目。

### 目标用户
| 用户 | 角色 | 需求 |
|------|------|------|
| **学员**（8-18岁） | 被辅导者 | 在引导下完成创新项目，获得反馈 |
| **教练/老师**（你） | 辅导者 | 管理多个学员项目，追踪进度，AI 辅助教学 |
| **评审**（ICC 评委） | 评估者 | 查看项目成果（间接） |

### 与现有项目的关系
```
AIICC 平台（本平台）—— 辅导基础设施
    ├── 辅导项目 A：星星糖 StarCandy（SDG 2 零饥饿）
    ├── 辅导项目 B：自学星球（自主学习力文具）
    └── 未来更多 ICC 参赛项目...
```

## 2. 核心功能

### 2.1 学员端 — 项目辅导工作流

学员进入平台后，按阶段推进：

#### Phase 1：发现 Discover
| 步骤 | AI 角色 | 学员产出 |
|------|---------|---------|
| 1.1 破冰 | 苏格拉底 | 说出想解决的问题 |
| 1.2 问题深挖 | 苏格拉底 | 5Why 分析树 |
| 1.3 SDG 对齐 | 分析师 | SDG 映射 + 选题方向 |
| 1.4 同理心 | 分析师 | 用户画像 + 痛点描述 |

#### Phase 2：定义 Define
| 步骤 | AI 角色 | 学员产出 |
|------|---------|---------|
| 2.1 价值主张 | 分析师 | 价值画布 |
| 2.2 竞品分析 | 分析师 | 竞品矩阵 |
| 2.3 创新定位 | 分析师 | 独特价值声明 |
| 2.4 项目立项 | 分析师 | 项目章程 |

#### Phase 3：构思 Ideate
| 步骤 | AI 角色 | 学员产出 |
|------|---------|---------|
| 3.1 头脑风暴 | 创意伙伴 | 至少 10 个创意 |
| 3.2 创意筛选 | 创意伙伴 | 筛选矩阵 |
| 3.3 方案细化 | 创意伙伴 | 方案描述 |
| 3.4 创新检查 | 审查官 | 创新性评估 |

#### Phase 4：原型 Prototype
| 步骤 | AI 角色 | 学员产出 |
|------|---------|---------|
| 4.1 MVP 定义 | 架构师 | MVP 范围 |
| 4.2 原型设计 | 架构师 | 线框图 / 故事板 |
| 4.3 技术方案 | 架构师 | 技术选型 |
| 4.4 验证计划 | 架构师 | 验证 checklist |

#### Phase 5：交付 Deliver
| 步骤 | AI 角色 | 学员产出 |
|------|---------|---------|
| 5.1 实现辅导 | 架构师 | 代码 / 产品 |
| 5.2 质量审查 | 审查官 | 审查报告 |
| 5.3 展示准备 | 审查官 | 演示文稿提纲 |
| 5.4 作品提交 | 审查官 | 提交 checklist |

### 2.2 AI 角色系统

| 角色 | 人设 | 擅长 | 语气 |
|------|------|------|------|
| **苏格拉底** | 爱提问的哲学家 | 引导思考、深挖问题 | 反问为主，不直接给答案 |
| **分析师** | 理性思维者 | 数据分析、逻辑推理 | 结构化、清晰 |
| **创意伙伴** | 天马行空的艺术家 | 发散思维、联想 | 热情、鼓励 |
| **架构师** | 经验丰富的工程师 | 方案设计、技术判断 | 务实、专业 |
| **审查官** | 严谨的评审 | 质量检查、风险评估 | 挑剔但建设性 |

### 2.3 教练端 — 管理后台

| 功能 | 说明 |
|------|------|
| 学员管理 | 查看所有学员及项目状态 |
| 进度总览 | 每个学员处于哪个阶段 |
| 项目看板 | 项目的关键里程碑 |
| 对话回放 | 查看学员与 AI 的对话记录 |
| 干预入口 | 在关键节点手动介入 |
| 模板管理 | 自定义辅导工作流模板 |

### 2.4 方法论引擎

| 方法论 | 适用阶段 | 工具 |
|--------|---------|------|
| 心光微笑模型 | 全过程 | 黄金圈、价值画布 |
| 设计思维 | Phase 1-3 | 同理心地图、HMW |
| 精益创业 | Phase 4-5 | MVP、验证学习 |
| 社会创新框架 | Phase 1-2 | SDG 映射、利益相关者分析 |

## 5. 与 Claude CLI 协同的关系

AIICC 平台 ≠ 替代 Claude CLI，而是互补：

| 场景 | 用 AIICC 平台 | 用 Claude CLI |
|------|--------------|---------------|
| 学员自助探索 | ✅ 结构化引导 | ❌ 需要教练陪伴 |
| 代码审查 | ❌ AI 审查质量不足 | ✅ Codex + Claude |
| 复杂架构讨论 | ❌ 深度不够 | ✅ 教练亲自把关 |
| 日常辅导 | ✅ 7x24 可用 | ❌ 教练时间有限 |
| 最终交付品 | ❌ 需要人工把控 | ✅ 教练兜底 |
